**Дата:** 08.04.2026
**Проект:** MyCash — учёт личных финансов в Telegram

---

## 1. Конкуренты: Telegram-боты и Mini App для учёта финансов

### 1.1 @wallet (Telegram Wallet)
- **Что делает:** Криптокошелёк внутри Telegram. Хранение, отправка, получение крипто (TON, USDT, BTC). P2P-обмен.
- **Формат:** Полноценный Mini App
- **Сильные стороны:** Официальная интеграция с Telegram, минималистичный UI, мгновенные переводы
- **Слабые стороны:** Только крипто, нет учёта расходов, нет категорий и отчётов
- **Что мы можем лучше:** У нас фиатные деньги (рубли), категории, дашборд, голосовой ввод

### 1.2 @CoinKeeperBot (CoinKeeper)
- **Что делает:** Быстрый учёт расходов через чат. Связан с мобильным приложением CoinKeeper.
- **Формат:** Текстовый бот (не Mini App)
- **Цена:** Бесплатно, Premium от 149 ₽/мес
- **Пользователи:** Миллионы скачиваний (приложение), бот менее популярен
- **Сильные стороны:** Быстрый ввод, категории, синхронизация с приложением
- **Слабые стороны:** Бот — только текст, без графического интерфейса. Полный функционал — в платном приложении
- **Что мы можем лучше:** Полноценный Mini App с красивым интерфейсом, голосовой ввод, дашборд прямо в Telegram

### 1.3 @ZenMoneyBot (Дзен-мани)
- **Что делает:** Учёт финансов с синхронизацией банковских счетов, бюджеты, отчёты
- **Формат:** Текстовый бот (основной функционал в веб/мобильном приложении)
- **Цена:** Бесплатно с ограничениями, Pro — от 2 990 ₽/год
- **Пользователи:** 1 000 000+ скачиваний на Google Play
- **Сильные стороны:** Мощный функционал, интеграция с банками
- **Слабые стороны:** Бот — дополнение к приложению, сложный для новичков, дорогой Pro
- **Что мы можем лучше:** Простота, не нужно скачивать приложение, всё внутри Telegram, бесплатно

### 1.4 @MoneyTrackerRobot
- **Что делает:** Запись расходов/доходов, категории, графики, экспорт в CSV
- **Формат:** Текстовый бот
- **Цена:** Бесплатно + Premium
- **Сильные стороны:** Графики, экспорт данных
- **Слабые стороны:** Текстовый интерфейс, нет Mini App
- **Что мы можем лучше:** Визуальный интерфейс, голос, дашборд

### 1.5 @cashbot_me (CashBot)
- **Что делает:** Ввод расходов текстом и голосом, мультивалютность, совместный учёт (семья)
- **Формат:** Текстовый бот
- **Цена:** Бесплатно + расширенный платный
- **Сильные стороны:** Совместный доступ, мультивалютность, голосовой ввод
- **Слабые стороны:** Только текстовый чат, нет графического интерфейса
- **Что мы можем лучше:** Mini App с полноценным UI, дашборд с диаграммами, два кошелька

## 4. Лучшие практики UX для Telegram Mini App

### Обязательно:
- `tg.ready()` и `tg.expand()` при запуске
- **MainButton** для главного действия ("Сохранить", "Записать")
- **BackButton** для навигации между экранами
- **HapticFeedback** — вибрация при нажатии кнопок и подтверждении
- Минимальный размер кнопок **44x44px**
- Поддержка **тёмной темы** (CSS-переменные из themeParams)
- **Skeleton/спиннер** при загрузке
- Сохранение состояния (localStorage / CloudStorage)

### Что бесит пользователей (антипаттерны):
1. Долгая загрузка (>2-3 секунды)
2. Белый экран без прелоадера
3. Мелкие кнопки и текст
4. Нет обратной связи при нажатии
5. Сломанная кнопка "Назад"
6. Запрос данных, которые уже есть в Telegram
7. Не сохраняется состояние при закрытии
8. Слишком много шагов до цели

### Что стоит добавить в MyCash:
- [ ] MainButton Telegram (нативная кнопка внизу)
- [ ] BackButton для навигации
- [ ] HapticFeedback при добавлении операции
- [ ] Поддержка тёмной темы через CSS-переменные Telegram
- [ ] Нативные попапы вместо confirm() — `tg.showPopup()`, `tg.showConfirm()`
- [ ] Подтягивание имени пользователя из Telegram
- [ ] Skeleton при первой загрузке

## 7. Экспертная оценка

### UX-дизайнер (оценка: 7.5/10)

**Ошибки новичков в финансовых приложениях:**
- Слишком много полей при вводе — человек хочет записать расход за 3 секунды, а не за 30
- Перегруженный главный экран — всё сразу: баланс, график, операции, кнопки
- Отсутствие онбординга — пользователь видит пустой экран и не понимает что делать
- Сложная система категорий (30+ штук) — оптимально 8-12

**Что обязательно:**
- Ввод за 3 тапа максимум (сумма → категория → сохранить)
- Баланс виден сразу при открытии, крупным шрифтом
- Редактирование и удаление операций
- Перевод между кошельками (снял наличные = не расход, а перемещение)

**Что бесит пользователей:**
- Забыл записать — статистика врёт, мотивация падает
- Принудительная категоризация (нет категории "Другое")
- Медленная загрузка (>2-3 сек)
- Невозможность отменить действие
- Навязчивая монетизация

**Рекомендации для MyCash:**
- Добавить редактирование операций (сейчас только удаление)
- HapticFeedback при сохранении — вибрация = "записано!"
- MainButton Telegram вместо HTML-кнопки
- Быстрые шаблоны: "Кофе — 250", "Обед — 500" — один тап
- Повторяющиеся операции (аренда, подписки — автозапись)
- Бюджеты по категориям с прогресс-баром
- НЕ делать привязку банков, НЕ делать авторизацию, НЕ перегружать первую версию

### Маркетолог (оценка: 6.5/10)

**Почему выберут MyCash, а не банк:**
- Банк видит только одну карту одного банка. MyCash — карту + наличку = полная картина
- 15-20% платежей в России наличные — банк их не видит
- Нулевой барьер входа — не нужно скачивать, нажал кнопку в Telegram и готово
- Голосовой ввод — 3 секунды вместо 30

**Конкретный месседж для рекламы:** "Сколько ты реально тратишь? Банк видит только карту. MyCash видит всё — за 3 секунды голосом."

**Удержание (главная проблема — 70-80% бросают в первые 2 недели):**
- Ежедневное напоминание в 21:00 прямо в Telegram-чат (не push, а сообщение)
- Мини-отчёт через 7 дней: "За неделю потратил X, больше всего на еду"
- Streak (серия дней подряд): "14 дней подряд записываешь!"
- Шаблоны частых трат — один тап вместо полного ввода

**Как продвигать:**
- Telegram-каталоги Mini Apps (TGStat, Telegstore)
- Посевы в каналах про финансы (T-J Финансы, "Экономим с умом") — GIF с голосовым вводом
- Рефералка: "Пригласи друга — расширенная статистика на месяц"
- Короткие видео в Reels/Shorts/TikTok — "до/после"
- Челлендж "30 дней учёта" в отдельном Telegram-канале
- Микро-инфлюенсеры (10-50К подписчиков) — 3000-10000 руб. за интеграцию

**Конверсия free→paid:**
- Триггер через 2-3 месяца: "У тебя данные за 3 месяца. Хочешь тренд? Это в Pro"
- "MyCash Бизнес" как отдельный продукт (не "Premium")
- Цена: 199-299 ₽/мес (ниже CoinKeeper, копейки для предпринимателя)
- Пробный Pro на 14 дней автоматически через месяц использования

**Слабые места:**
- Конкуренция с бесплатной аналитикой банков
- Люди бросают вести учёт — главный враг не конкуренты, а лень
- Только 3-5% пользователей в РФ готовы платить за приложение
- Голосовой ввод бесплатно = расходы на распознавание при масштабе

**Пик интереса:** Январь ("новая жизнь") и сентябрь (после отпусков)

### Telegram-разработчик (оценка: 6/10)

**Что работает хорошо:**
- Однофайловая структура — для MVP нормально
- Apple-стиль дизайна — грамотно
- Голосовой парсер — продуман, поддержка числительных, категорий
- Демо-данные — отличное решение
- Круговая диаграмма на Canvas — лёгкая, без библиотек

**Что проблема:**
- `confirm()` может не работать в Mini App — заменить на `tg.showConfirm()`
- Удаление по клику — опасный UX, случайный тап вызывает удаление
- `Date.now()` как ID — коллизии при быстром вводе
- Нет валидации данных из localStorage
- Баланс считается как разность за период, а не текущий остаток на счетах
- Нет начальных остатков кошельков
- CloudStorage подключен формально, но Storage.load всегда читает из localStorage

**Технические рекомендации (критичные):**
1. Заменить `confirm()` на `tg.showConfirm()`
2. Добавить `tg.enableClosingConfirmation()` при открытой форме
3. Исправить ID: `Date.now().toString(36) + Math.random().toString(36).substr(2,5)`
4. Валидация данных при загрузке
5. Начальные остатки кошельков + текущий баланс (а не за период)

**Технические рекомендации (важные):**
6. Исправить Storage.load — реальное чтение из CloudStorage
7. Шардировать данные по месяцам для CloudStorage (лимит 4096 байт/ключ)
8. HapticFeedback: `impactOccurred('light')` при нажатии, `notificationOccurred('success')` при сохранении
9. Экспорт данных (хотя бы JSON для бэкапа)
10. `tg.themeParams` для поддержки тёмной темы

**Архитектура при 100+ пользователях:**
- Mini App на GitHub Pages — выдержит (это CDN)
- localStorage — каждый хранит у себя, сервер не нагружается
- Бот на компьютере — ПРОБЛЕМА, нужен VPS для 24/7
- Главный риск: потеря данных пользователей при очистке кеша Telegram
